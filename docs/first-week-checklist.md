# First Week Checklist — первый клиент LogiRoute

> **Для кого:** директор, диспетчер, кладовщик, owner LogiRoute (внедрение).  
> **Не для разработчика.** Цель: к концу недели клиент **работает без Excel** на ежедневных маршрутах.

**Рекомендуемый пилот:** 1 компания · 3–5 водителей · 50–100 клиентов · 100–300 доставок/день.

**Роли LogiRoute на старте:** super_admin создаёт компанию; admin/owner — пользователи; вы сопровождаете день 1–2.

---

## День 0 (до понедельника — подготовка)

- [ ] Подписан договор / оплата (plan **full** или **ops** — по договорённости)
- [ ] Получены: Excel клиентов, список водителей (имя, телефон, email), координаты/адрес склада
- [ ] Назначен **один ответственный на стороне клиента** (обычно диспетчер)
- [ ] У водителей: Android-телефон или браузер; включена геолокация

---

## День 1 — «Машины могут поехать»

| # | Задача | Кто | Готово |
|---|--------|-----|--------|
| 1 | Создана компания, trial/active billing | LogiRoute super_admin | ☐ |
| 2 | Создан **owner** или **admin** клиента | LogiRoute | ☐ |
| 3 | Создан **dispatcher** | Admin клиента | ☐ |
| 4 | Созданы **водители** (3–5) | Admin | ☐ |
| 5 | **Импорт клиентов** (Excel) | Dispatcher + LogiRoute | ☐ |
| 6 | Проверка geocode: 5–10 ключевых адресов на карте | Dispatcher | ☐ |
| 7 | **Координаты склада** заданы (dispatcher menu) | Dispatcher | ☐ |
| 8 | **Импорт / ввод товаров** (если склад) или справочник מק"ט | Кладовщик / admin | ☐ |
| 9 | **Первые точки доставки** (5–10 шт.) — вручную или import | Dispatcher | ☐ |
| 10 | **Первый маршрут** создан, водитель назначен | Dispatcher | ☐ |
| 11 | Водитель: login, видит точки, **тестовая** доставка (1 точка) | Водитель | ☐ |
| 12 | **Первый счёт / накладная** (если full) — 1 документ на тестовую точку | Dispatcher / бухгалтер | ☐ |

**Критерий конца дня 1:** хотя бы **1 реальная или учебная доставка** закрыта в системе (status completed).

---

## День 2 — «Ежедневная работа»

| # | Задача | Готово |
|---|--------|--------|
| 1 | Полный день: точки → маршруты → все водители в поле | ☐ |
| 2 | Корректировка порядка остановок / смена водителя | ☐ |
| 3 | Склад: остатки сверены с реальностью (хотя бы топ-10 SKU) | ☐ |
| 4 | **Отчёт:** admin analytics или owner overview — «сколько доставлено» | ☐ |
| 5 | Разбор проблем (список ≤ 5 пунктов) с LogiRoute | ☐ |

**Критерий:** диспетчер **не вёл параллельный маршрут в Excel**.

---

## День 3 — «Склад и привычка»

| # | Задача | Готово |
|---|--------|--------|
| 1 | Склад: приход/расход или инвентаризация (частичная) | ☐ |
| 2 | Повтор импорта точек (если есть утренний файл заказов) | ☐ |
| 3 | Второй диспетчер (backup) — 2 ч обучения | ☐ |
| 4 | POD / фото (если включено в настройках компании) | ☐ |

---

## День 4–5 — «Стабилизация»

- [ ] Все водители закрывают точки **в приложении** (не звонком диспетчеру)
- [ ] Счета за период — **≥ 80%** через LogiRoute (если full)
- [ ] Зафиксированы 3 главных UX-проблемы → в backlog «первый клиент»

---

## Конец недели 1 — «Первая сверка»

| # | Проверка | ☐ |
|---|----------|---|
| 1 | Количество completed в системе ≈ факт доставок (±5%) | |
| 2 | Нет «потерянных» маршрутов (все водители видели свои точки) | |
| 3 | Клиент знает, к кому звонить при сбое (контакт support) | |
| 4 | Решено: продолжаем пилот на месяц 2 (см. pilot-success-criteria) | |

---

## Типичные причины срыва недели 1 (не код)

- Импорт клиентов без проверки адресов → водитель «не находит»
- Нет человека на стороне клиента с полномочиями решать
- Водители не прошли 30-мин инструктаж
- Ожидание «всё как в старой программе» без готовности сменить привычку
