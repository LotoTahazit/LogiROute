# סוגי מסמכים ומספור רץ

> **מסמך זה נדרש להגשה לרישום תוכנה לניהול ספרים (מס הכנסה)**

## סוגי מסמכים נתמכים

המערכת תומכת בסוגי המסמכים הבאים:

| סוג מסמך | קוד פנימי | אוסף Firestore | סדרת מספור |
|-----------|-----------|----------------|------------|
| חשבונית מס | `tax_invoice` | `accountingDocs` | סדרה נפרדת |
| קבלה | `receipt` | `accountingDocs` | סדרה נפרדת |
| חשבונית מס קבלה | `tax_invoice_receipt` | `accountingDocs` | סדרה נפרדת |
| תעודת זיכוי | `credit_note` | `accountingDocs` | סדרה נפרדת |

## מחזור חיי מסמך (State Machine)

```
draft → issued → locked → credited
                        → voided_before_delivery (מ-draft בלבד)
```

| סטטוס | ערך | תיאור |
|--------|------|--------|
| טיוטה | `draft` | ניתן לעריכה, לא מקבל מספר |
| הונפק | `issued` | קיבל מספר + hash, לא ניתן לשינוי |
| נעול | `locked` | נעול לתקופת חשבונאות |
| זוכה | `credited` | בוטל על ידי תעודת זיכוי |
| בוטל לפני מסירה | `voided_before_delivery` | בוטל לפני מסירה ללקוח (מ-draft בלבד) |

## מספור רץ

### עקרונות
- לכל סוג מסמך סדרת מספור נפרדת ועצמאית
- המספור רציף, ללא דילוגים וללא כפילויות
- הקצאת מספר מתבצעת באופן אטומי (Firestore Transaction)
- מספר מוקצה רק בעת מעבר `draft → issued` (לא בעת יצירת טיוטה)

### מנגנון טכני

**נתיב מוני המספור:**
```
companies/{companyId}/accounting/_root/counters/{docType}
```

**שדה המונה:** `lastNumber` (int)

**תהליך הקצאת מספר (בתוך `issueDoc()` transaction):**
1. קריאת המונה הנוכחי
2. חישוב `nextNumber = lastNumber + 1`
3. עדכון המונה: `lastNumber = nextNumber`
4. כתיבת `docNumber`, `issuedAt`, `immutableSnapshotHash` למסמך
5. עדכון `status → issued`

כל 5 הפעולות מתבצעות בתוך `FirebaseFirestore.runTransaction()` אחד — אטומי.

**Firestore Security Rules — אכיפת increment:**
```javascript
// counters: strictly +1
match /accounting/_root/counters/{counterId} {
  allow create: if ... && request.resource.data.lastNumber == 1;
  allow update: if ... && request.resource.data.lastNumber == resource.data.lastNumber + 1;
  allow delete: if false;
}
```

> **ההגנה המשפטית היא בצד השרת**: גם אם לקוח זדוני ינסה לדלג על מספר — Firestore Rules ידחו את הבקשה.

### בדיקת שלמות מספור

פונקציה `verifySequentialIntegrity()` בודקת לכל סוג מסמך:
- רציפות המספרים (1, 2, 3, ...)
- היעדר כפילויות
- היעדר פערים

### דוגמה
```
חשבונית מס:        1, 2, 3, 4, 5...
קבלה:              1, 2, 3...
חשבונית מס קבלה:  1, 2, 3, 4...
תעודת זיכוי:      1, 2...
```

## מחיקה

**מחיקת מסמכים אסורה** — Firestore Security Rules:
```javascript
allow delete: if false;
```

בנוסף, `deleteDoc()` ב-`AccountingDocsRepository` תמיד זורק `UnsupportedError`:
```dart
Future<void> deleteDoc(String docId) async {
  throw UnsupportedError(
    'Deletion of accounting documents is forbidden (ניהול ספרים). '
    'Use createCreditNote() for corrections or voidBeforeDelivery() '
    'for cancellation before delivery.',
  );
}
```

ביטול מסמך שכבר הונפק מתבצע על ידי הפקת **תעודת זיכוי** (`credit_note`) — מסמך חדש עם קישור למסמך המקורי.

ביטול לפני מסירה (`voided_before_delivery`) — רק ממצב `draft`, לפני הנפקה.
