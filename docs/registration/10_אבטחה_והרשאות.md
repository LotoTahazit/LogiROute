# אבטחה והרשאות

> **מסמך זה נדרש להגשה לרישום תוכנה לניהול ספרים (מס הכנסה)**

## אימות (Authentication)

- **ספק**: Firebase Authentication
- **שיטות**: אימייל + סיסמה
- **טוקן**: JWT — מאומת בכל בקשה ל-Firestore
- **session**: מנוהל אוטומטית על ידי Firebase SDK

## הרשאות (Authorization)

### תפקידים

| תפקיד | קוד | הרשאות |
|--------|------|---------|
| סופר-אדמין | `super_admin` | גישה מלאה לכל החברות |
| בעלים | `owner` | קריאה בלבד לדוחות, ניהול חברי צוות |
| מנהל | `admin` | ניהול מלא של חברה ספציפית |
| רואה חשבון | `accountant` | יצירה/עדכון מסמכים חשבונאיים (draft בלבד), קריאת דוחות ויומן ביקורת |
| מוקדן | `dispatcher` | הפקת מסמכים, ניהול מסלולים |
| נהג | `driver` | צפייה במסלול, עדכון סטטוס משלוח |
| מחסנאי | `warehouse_keeper` | ניהול מלאי |

> **תפקיד `accountant`** — תפקיד ייעודי לרואה חשבון. גישה מוגבלת למודול החשבונאות בלבד:
> - קריאה: `accounting`, `reports`, `audit`
> - כתיבה: `accountingDocs` בלבד (create/update, לא delete)
> - אין גישה ללוגיסטיקה, מחסן, ניהול חברה

## Firestore Security Rules — עקרונות

1. **כל בקשה דורשת אימות**: `request.auth != null`
2. **הפרדת דיירים**: `userDoc().companyId == companyId`
3. **Billing guard**: גישה רק לחברות פעילות (`active`, `grace`, `trial`)
4. **Module guard**: גישה רק למודולים מופעלים (`modules.accounting == true`)
5. **append-only**: יומן ביקורת, שרשרת שלמות, עוגני שלמות
6. **אי-מחיקה**: מסמכים חשבונאיים, מוני מספור
7. **אי-שינוי**: `docNumber`, `issuedAt`, `immutableSnapshotHash` לאחר הנפקה
8. **אטומיות מספור**: `lastNumber == resource.data.lastNumber + 1`
9. **זיהוי שחקן**: `createdBy == request.auth.uid`

### Firestore Rules — accountingDocs (עיקרי)
```javascript
match /accountingDocs/{docId} {
  allow read: if isCompanyMember(companyId);
  allow create: if isCompanyMember(companyId)
    && hasAnyRole(['accountant', 'admin', 'super_admin'])
    && companyIdMatchesPath(companyId)
    && request.resource.data.createdBy == request.auth.uid;
  allow update: if isCompanyMember(companyId)
    && hasAnyRole(['accountant', 'admin', 'super_admin'])
    && unchanged('createdAt') && unchanged('createdBy') && unchanged('companyId')
    && (hasAnyRole(['admin', 'super_admin']) || resource.data.status == 'draft')
    && (!('docNumber' in resource.data) || resource.data.status == 'draft' || unchanged('docNumber'))
    && (!('issuedAt' in resource.data) || resource.data.status == 'draft' || unchanged('issuedAt'))
    && (!('immutableSnapshotHash' in resource.data) || resource.data.status == 'draft' || unchanged('immutableSnapshotHash'));
  allow delete: if false;
}
```

### Firestore Rules — counters (מוני מספור)
```javascript
match /accounting/_root/counters/{counterId} {
  allow read: if canUseModule(companyId, 'accounting');
  allow create: if canWriteModule(companyId, 'accounting')
    && hasAnyRole(['admin', 'super_admin'])
    && request.resource.data.lastNumber == 1;
  allow update: if canWriteModule(companyId, 'accounting')
    && request.resource.data.lastNumber == resource.data.lastNumber + 1;
  allow delete: if false;
}
```

## הגנות נוספות

### הגנה בצד לקוח (UX)
- `ImmutabilityGuard` — מונע שליחת בקשות שיידחו
- הודעות שגיאה בעברית
- חסימת כפתורים לפי סטטוס

### הגנה בצד שרת (Firestore Rules)
- **הערבות המשפטית** — Firestore Security Rules
- הלקוח (ImmutabilityGuard) הוא רק UX
- גם אם הלקוח נפרץ — השרת חוסם שינויים אסורים

### HTTPS
- כל התקשורת מוצפנת (Firebase default, TLS 1.2+)

### יומן גישה
- כל פעולה חשבונאית נרשמת ב-`companies/{companyId}/audit/{eventId}`
- append-only — לא ניתן למחוק
