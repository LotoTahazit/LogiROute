## סיכום עמידה בדרישות

### דרישות חוק ניהול ספרים

| דרישה | סטטוס | פירוט |
|--------|--------|--------|
| מספור רץ רציף | ✅ | סדרות נפרדות, אטומי, ללא פערים |
| אי-מחיקה | ✅ | `allow delete: if false` |
| אי-שינוי לאחר סיום | ✅ | Firestore Rules + SHA-256 hash |
| יומן ביקורת | ✅ | append-only, log-before-action |
| הדפסת מקור אחד | ✅ | חסימה אוטומטית |
| סימון עותקים | ✅ | מקור / עותק / נאמן למקור |
| סימון טיוטה | ✅ | "טיוטה — לא לשימוש רשמי" |
| סימון הדפסה חוזרת | ✅ | תאריך ושעה |
| גיבוי רבעוני | ✅ | רישום + בדיקת שחזור |
| קובץ במבנה אחיד | ✅ | CSV עם מטא-נתונים |
| שמירה 7+ שנים | ✅ | בדיקה תקופתית |
| הגנה מפני כשל חשמל | ✅ | Firestore ACID transactions |
| שלמות שעון | ✅ | serverTimestamp — שעון לקוח לא משפיע |
| ביטול עם סיבה | ✅ | סיבת ביטול חובה — המערכת אוכפת |
| אחסון גיבוי מוצפן | ✅ | AES-256, אזור me-west1 (ישראל) |

### דרישות חשבוניות ישראל

| דרישה | סטטוס | פירוט |
|--------|--------|--------|
| מספר הקצאה | ✅ | בקשה אוטומטית לאחר סיום |
| ספי חובה | ✅ | 20K/10K/5K לפי תאריך |
| חסימת הדפסה ללא הקצאה | ✅ | מקור חסום עד אישור |
| ריטריי + דדופליקציה | ✅ | exponential backoff, 3 ניסיונות |
| רישום סירובים | ✅ | יומן ביקורת + assignment_requests |
| ניסיון חוזר ידני | ✅ | manual retry עם רישום מלא |

## הערות לתהליך ההגשה

1. **הגשה מקוונת**: דרך אתר gov.il — שירות רישום תוכנה לניהול ספרים
2. **עלות**: חינם
3. **זמן טיפול**: עד 90 יום מהגשה מלאה
4. **סימולטור**: יש להריץ את קובץ הייצוא דרך סימולטור רשות המסים ולצרף את הדוח
5. **דוגמאות פלט**: יש להפיק דוגמאות PDF של כל סוגי המסמכים ולצרף

## Roadmap — סדר פעולות להגשה

1. ✅ הקפאת מסמכים: draft → final (immutable), credit note
2. ✅ מספור: סדרות + אטומי + בדיקת רציפות
